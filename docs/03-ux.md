# Tier 3 - User Experience

Brings the UI to a polished, daily-use standard.

---

## 3.1 Per-member points on Leaderboard tab [todo]

**Problem:** The Leaderboard tab currently shows all members with 0 points because member-level totals aren't computed. House-level scores work; individual scores don't.

**Approach:**
Add `POST /users/scores` to the API; it sums `PointTransaction.delta` grouped by `actorUserId` within the org. Return `[{ userId, displayName, houseId, houseColor, houseName, totalPoints }]` sorted desc.

Wire this into `readLeaderboard`-style server action, pass to the `Leaderboard` component replacing the placeholder `memberPoints` array.

---

## 3.2 Optimistic score update after awarding points [todo]

**Problem:** After submitting the Award Points dialog, the house score only updates after a full page reload triggered by `revalidatePath`.

**Approach:**
- Add a `onSuccess` callback to `AwardPointsDialog` that accepts the `{ targetHouseId, delta }` 
- In `DashboardShell`, maintain a local `Map<houseId, scoreOffset>` in state
- Apply the offset to the displayed score immediately, clearing it on next data fetch
- This is a cosmetic optimistic update; the real value comes back on revalidation

---

## 3.3 Activity feed pagination [todo]

**Problem:** `POST /transactions/recent` is hard-capped at 50 records. Long-running orgs will lose history.

**Approach:**
- Add `cursor` and `take` query params to the endpoint
- `ActivityFeed` component gets a "Load more" button that calls a new `readMoreActivity(cursor)` server action
- Append results to the existing list client-side

---

## 3.4 Admin: edit house color / description [todo]

**Problem:** Once a house is created, its color and description can't be changed through the UI. The API's `upsert` currently only sets them on create.

**Approach:**
- Update `POST /admin/houses` upsert to also update `color` and `description` on conflict
- Add an edit form to the admin section alongside the create form
- Use the same `createHouseSchema`; house `name` stays as the unique key

---

## 3.5 Mobile header [todo]

**Problem:** On narrow screens the header can get crowded (house badge + award button + icons).

**Approach:**
- Hide the house badge below `sm` (already done with `hidden sm:inline-flex`)
- Move "Award Points" to a floating action button (FAB) on mobile in the bottom-right corner
- The tab labels can collapse to icons-only below `sm`
